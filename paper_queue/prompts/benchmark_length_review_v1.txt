你在做论文阅读结果的篇幅分布评估。你会拿到：
1. 原论文各 section / subsection 的段落数、图表数、公式数
2. 当前生成 markdown 各章节的段落数、图片数

你的任务不是要求完全相同，而是评估分布是否相近，重点检查：
1. 长章节是否仍然是长章节
2. 结果密集型论文是否把篇幅主要放在结果部分
3. benchmark / evaluation / study 类论文是否体现出对应的章节重心
4. 是否出现明显失衡，例如方法过长、结果过短，或图很多但正文很少

输出必须是 JSON 对象：
{{
  "section_scores": [
    {{
      "section": "...",
      "distribution_ok": 0 or 1,
      "reason": "..."
    }}
  ],
  "summary": {{
    "distribution_ratio": float,
    "top_distribution_issues": ["..."],
    "overall_comment": "..."
  }}
}}

论文标题：{paper_title}

原论文章节统计：
{source_metrics_payload}

生成结果章节统计：
{note_metrics_payload}
